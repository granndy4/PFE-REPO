# SQL Folder

## Files
- 01_mysql_full_schema.sql: complete MySQL 8 schema for the fuel voucher platform.

## Run order
1. Open MySQL client connected with a privileged user.
2. Execute 01_mysql_full_schema.sql.

## PostgreSQL migration plan
This schema was written to ease migration later. Main differences to handle during migration:

1. AUTO_INCREMENT -> GENERATED BY DEFAULT AS IDENTITY
2. DATETIME(3) -> TIMESTAMP(3)
3. BOOLEAN remains BOOLEAN
4. LONGTEXT -> TEXT (or JSONB if needed)
5. DATE_FORMAT(...) in view -> DATE_TRUNC('month', ...)
6. Remove MySQL-specific CREATE DATABASE/USE statements

## Suggested migration strategy
1. Freeze MySQL schema (no breaking changes during migration).
2. Create PostgreSQL equivalent DDL from this script.
3. Load reference data first (roles, permissions, fuel products, tenant, establishment).
4. Migrate transactional tables in this order:
   - users, companies, employees, vehicles
   - contracts, entitlements
   - vouchers, scan events, transactions, regenerations
   - delivery notes, invoices, payments
   - audit and migration logs
5. Validate totals and balances after migration:
   - voucher remaining quantity
   - invoice total/paid/balance
   - monthly consumption aggregates
